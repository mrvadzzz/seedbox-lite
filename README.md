# Seedbox Lite

Локальный веб-интерфейс для добавления торрент-файлов и magnet-ссылок, просмотра видео во время загрузки и продолжения воспроизведения с другого устройства в домашней сети.

Проект основан на [hotheadhacker/seedbox-lite](https://github.com/hotheadhacker/seedbox-lite) и распространяется по некоммерческой лицензии из файла [LICENSE](LICENSE).

## Возможности

- добавление `.torrent`-файлов и magnet-ссылок;
- общая серверная история для компьютера, телефона и телевизора;
- потоковое воспроизведение с поддержкой HTTP Range;
- выбор аудиодорожки и преобразование несовместимого звука через FFmpeg;
- субтитры, запоминание позиции и полноэкранный режим;
- интерфейс для пульта телевизора и сенсорного экрана;
- запуск frontend и backend одной командой через Docker Compose.

## Требования

- Docker Desktop с Docker Compose;
- свободные порты `5174`, `3001`, `6881/TCP` и `6882/UDP`;
- не менее 2 ГБ свободной оперативной памяти на время работы;
- компьютер и устройство просмотра в одной локальной сети.

## Быстрый запуск

1. Создайте локальную конфигурацию.

   В PowerShell:

   ```powershell
   Copy-Item .env.example .env
   ```

   В Linux или macOS:

   ```bash
   cp .env.example .env
   ```

2. Откройте `.env` и обязательно замените пароль:

   ```env
   ACCESS_PASSWORD=change-this-password
   ```

3. Соберите и запустите сервис:

   ```powershell
   docker compose up --build -d
   ```

4. Откройте на компьютере:

   ```text
   http://localhost:5174
   ```

Проверка backend:

```text
http://localhost:3001/api/health
```

## Телефон и телевизор

Узнайте локальный IPv4-адрес компьютера командой:

```powershell
ipconfig
```

Затем откройте на телефоне или телевизоре адрес вида:

```text
http://192.168.0.173:5174
```

Подставьте адрес своего компьютера. Значение `VITE_API_BASE_URL` по умолчанию пустое, поэтому frontend автоматически обращается к backend на том же IP и порту `3001`. Задавать IP вручную в `.env` обычно не нужно.

Если страница не открывается, разрешите входящие подключения к портам `5174` и `3001` в брандмауэре Windows и убедитесь, что оба устройства подключены к одной сети. Динамический внешний IP провайдера для домашнего доступа не важен; используется локальный адрес вида `192.168.x.x`.

## Настройка

Основные параметры находятся в `.env`:

| Переменная | Назначение | Значение по умолчанию |
| --- | --- | --- |
| `ACCESS_PASSWORD` | Пароль для входа | нужно изменить |
| `FRONTEND_PORT` | Порт веб-интерфейса | `5174` |
| `BACKEND_PORT` | Порт API | `3001` |
| `TORRENT_PORT` | Входящий TCP-порт BitTorrent | `6881` |
| `DHT_PORT` | UDP-порт DHT | `6882` |
| `TORRENT_UPLOAD_LIMIT` | Ограничение отдачи в байтах/с | `262144` |
| `VITE_API_BASE_URL` | Явный адрес API, если frontend и backend находятся на разных хостах | пусто |
| `OMDB_API_KEY` | Необязательный ключ OMDb для метаданных | пусто |
| `TMDB_API_KEY` | Необязательный ключ TMDB для метаданных и фонов | пусто |

После изменения `VITE_API_BASE_URL` пересоберите frontend:

```powershell
docker compose up --build -d
```

## Управление

```powershell
# Состояние контейнеров
docker compose ps

# Журнал работы
docker compose logs -f

# Остановка без удаления истории и кэша
docker compose down

# Полная очистка данных приложения
docker compose down -v
```

Последняя команда удаляет Docker-тома `seedbox_data` и `seedbox_cache`, включая сохранённую историю и загруженные части файлов.

## Разработка без Docker

Нужны Node.js 18+ и установленный в системе FFmpeg.

```powershell
npm run install-all
Copy-Item .env.example server/.env
```

Задайте в `server/.env` `NODE_ENV=development`, затем запустите две команды в отдельных окнах терминала:

```powershell
npm run server
npm run client
```

## Данные и публикация

Пароли, torrent-файлы, кэш, загруженные данные, логи, `node_modules` и готовая сборка исключены через `.gitignore` и `.dockerignore`. В репозиторий следует добавлять только `.env.example`; файл `.env` публиковать нельзя.

Рабочие данные Docker хранятся в именованных томах и не копируются вместе с папкой исходников.

## Безопасность

Сервис рассчитан на доверенную домашнюю сеть. Не публикуйте порты `3001` и `5174` напрямую в интернет без HTTPS, обратного прокси и дополнительной защиты доступа.

Backend использует WebTorrent 1.x для совместимости с текущей CommonJS-реализацией. `npm audit` сообщает о шести предупреждениях высокого уровня в его старой цепочке сетевых зависимостей. Автоматическое исправление требует перехода на WebTorrent 2.x и отдельной переработки backend; применять `npm audit fix --force` без такой миграции не следует.

## Ограничения

- Доступность и скорость видео зависят от сидов и пиров.
- Некоторые телевизионные браузеры ограничивают нативный полноэкранный режим. Проигрыватель использует контейнерный полноэкранный режим, чтобы сохранить собственные элементы управления.
- Для MKV, AC3, E-AC3 и DTS может потребоваться преобразование через FFmpeg, поэтому переключение дорожки создаёт дополнительную нагрузку на компьютер.
